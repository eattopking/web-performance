6. react setState()做了哪些事情，流程是什么？
   首先创建一个更新对象，将更新对象放到更新队列中（更新队列是和组件唯一绑定的），然后根据优先级进行调度更新，（render阶段，在内存中完成）多个同时更新需要批处理得出最终的state，然后将state更新到虚拟dom中（创建一个新树新增和修改的节点在新树上体现标记、删除的节点在新树的deletions字读内存这个， 删除节点在老树节点上体现标记），（commit阶段，开始更新真实dom了）最后进行页面渲染（最后用新树进行页面渲染），然后调用生命周期

7. react中组件key的作用是什么?
   key就是虚拟dom元素的标识，提高diff算法的效率，降低更新的时间，保证更新的准确性

8. React Fiber架构的实现原理?React Fiber 是对 React核心算法的一次重大重构，引入可中断的渲染、优先级管理和更好的错误处理（任何未被捕获渲染错误，都会导致组件树被卸载， 错误处理可以使用react-error-boundary组件去捕获错误，捕获到错误之后可以重试）
9. useEffect 与 useLayoutEffect的区别? useEffect:在浏览器完成布局和绘制之后异步执行。useLayoutEffect:在所有DOM 变更之后、浏览器绘制之前同步执行
10. vue2跟vue3的区别有哪些
    编程范式的改变从选项式改为了组合式，选项式逻辑写在各个选项中不容易复用不够灵活， 组合式之后复用能力增强，可以提取hooks

    响应式实现原理改变了从Object.defineProperty改为了proxy，defineProperty的问题（无法拦截对象属性的新增和删除、无法完美监听数组索引和长度变化、 初始化需要递归遍历整个对象比较耗时），Proxy是惰性监听只有在访问到比较深层次的属性的时候才进行代理

    vue3 api按需引入，打包的体积更小， 虚拟dom更快了，因为编译阶段做大量的静态提升（就是把不会变的html节点在编译的时候提存储到缓存里，渲染的时候直接拿来用）和事件监听缓存（把模版里的事件监听替换放在缓存中，渲染的时候拿来用，每次都是一个不需要重新创建）优化渲染时间

    vue3使用ts开发， vue2使用js开发，vue3的ts支持程度更好

    新增了Suspense组件和react一样，可以展示异步加载的loading状态

11. Tree Shaking的原理
    Tree Shaking本质就是标记导出了但是没有被使用的代码，在代码编译阶段跟踪每个export出去的内容，在import的时候没被真实使用或者没有地方import它就标记为未使用，如果import并使用了就标记为使用，然后在代码压缩阶段删除没有被使用的代码，减小包的体积

12. Tree Shaking为什么必须使用esmodule，因为esmodule是静态的可预测、稳定，commonjs是动态不可预测、不稳定，删除代码有不确定性可能出现问题
13. node_module中的依赖会被vite和webpack编译吗，webpack默认会全部编译，但是可以配置过滤（不编译但是会分析， 也会进行tree shaking的操作）， vite开发环境会进行预编译（将模块化统一成esm满足浏览器esm模块、合并文件将多文件依赖合并成一个包方便请求）、vite生产环境也会编译（但是之后进行模块化更新、代码打包合并、代码压缩等）
14. rollup和webpack比的坏处是什么
    不支持热更新和启动本地服务无法进行前端应用开发、对commonjs、amd这种模块化支持的不够完善、对除了js的文件支持的也不够完善、代码分割支持的也不够完善、社区没有webpack庞大

15. webpack、vite、rspack有什么区别
    RsPack使用 Rust 语言重写了整个构建引擎。得益于 Rust 强大的多线程并发计算能力，它硬生生地用算力把打包速度提升了上来，生态还没有那么好

    vite 是预构建、按需编编译、生成环境使用rollup

    webpack先构建再启动，js实现单线程打包构建速度慢，兼容性强（文件兼容性、模块化兼容性）、生态好

16. HTTPS ssI建立流程
    首先使用客户端和服务端使用非对称加密的方式传递消息确认后续通信时所用的对称加密的密钥内容，并建立连接，然后使用对称加密进行通信

    首先客户单给服务端发消息想要确认对称加密的密钥，服务端收到信息后，给客户端返回证书（包含公钥），客户端将生成的对称加密密钥用服务端返回的公钥加密后，发给服务端，服务端收到后解密获取到对称加密的密钥，然后通过对称加密通信
